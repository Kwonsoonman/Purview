# 작업 3: 메시지 암호화 검증
이 작업에서는 재무 부서 직원으로부터 테스트 이메일을 보내 Microsoft Purview 메시지 암호화가 자동으로 적용되었는지, 수신자가 보안 메시지 알림을 보고 있는지 확인합니다.

<br>
⚠️ 주의: 일부 실험실 환경에서는 외부 이메일 전달이 차단될 수 있습니다. 이 작업이 예상대로 완료되지 않을 수도 있습니다.


<br>
1.	작업 표시줄에서 Microsoft Edge를 우클릭한 후 New InPrivate 창을 선택하여 Microsoft Edge를 InPrivate 창에서 엽니다.
<br>
2.	https://outlook.office.com 로 이동해 Outlook에 로그인하세요 (LynneR 계정)
<br>

<br>
3.	웹의 Outlook에서 새 메일을 작성합니다. <br>

<br>
4.	받은 사람에 개인 또는 제3자 이메일 주소를 입력하고,

+ 제목: 비밀 메시지
+ 이메일 본문: 'My super-secret message.'를 입력한 후 메시지를 전송합니다.

<br>

<br>
5.	개인 이메일 계정에 새 창에서 로그인하고 린 로빈스가 보낸 메시지를 열어보세요. 만약 메시지를 Microsoft 계정(예: @outlook.com)으로 보냈다면, 자동으로 열릴 수도 있습니다. 만약 (@gmail.com) 같은 다른 이메일 서비스에 이메일을 보냈다면, 암호화를 처리하고 메시지를 읽기 위해 다음 단계를 수행해야 할 수도 있습니다.
<br>
<br>
6.	메시지를 읽으면, 한정 시간 비밀번호를 받으려면 일회용 비밀번호로 로그인을 선택합니다.<br>

<br>
7.	개인 이메일 포털에 접속해 제목인 '일회용 비밀번호'로 메시지를 열어 메시지를 확인하세요.
<br>

<br>
8.	비밀번호를 복사해서 포털에 붙여넣고 계속을 선택하세요.<br>

<br>
9.	암호화된 메시지를 검토하세요. 이메일 하단에 '이 이메일은 암호화되어 Contoso Ltd.에서 안전하게 발송되었습니다'라는 메시지를 보실 수 있습니다. 재무 부서에서 온 메시지가 자동으로 암호화되고 Contoso 면책 조항이 포함되어 있어 Microsoft Purview 메시지 암호화가 정상적으로 작동하고 있음을 확인하셨습니다.
